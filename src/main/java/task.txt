CRUD - Create Read Update Delete
Разработать приложение, которое позволит хранить различную информацию о пользователях.
1. Приложение должно позволять просматривать, добавлять, изменять, удалять пользователей, каждый пользователь уникален
2. каждый пользователь может иметь 0..* город , где работал
3. каждый пользователь может иметь 1..* городов где жил
4. должна быть возможность посмотреть всех пользователей которые работали в городе X
5. должна быть возможность посмотреть всех пользователей которые в городе Y в городе X
6. пункты 5 и 6 должны работать совместно

ограничения:
-Любая удобная тебе реляционная, база данных
-запрещено использовать любой фреймворк который облегчает работу с базой данной, можно юзать только нативный SQL и конекторы к БД

Используем БД PostgreSQL
JDBC -- драйвер(Библиотека классов) в джаве необходимый для работы с БД
Программа будет иметь слоистую архитектуру(данные ходят от слоя к слою, стандартно)
Главные три слоя:
1. Слой контроллера отвечает за принятие реквеста(запроса) и определяет какой метод сервиса вызвать. Так же
он совершает валидацию(проверка данных на правильность)
2. Слой сервиса -- вся бизнес-логика программы, работа над данными
3. Репозиторий/DAO -- работа с БД (методы работающие с БД)

Слои изолированы! Слой видит только соседа сверху и снизу
Каждый слой находится в своем package
